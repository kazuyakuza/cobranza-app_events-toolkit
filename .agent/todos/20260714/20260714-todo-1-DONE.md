# Fix: @cobranza-apps/events-toolkit — OnEventExplorer still crashes on getter properties

## Background

Version `0.10.6` attempted to fix the `OnEventExplorer` crash by adding a `typeof methodRef !== 'function'` guard, but the fix is incomplete. The crash now happens at a different point.

**Confirmed affected consumer:** `ms-db-gateway`.

## Error

```
TypeError: Cannot read properties of undefined (reading 'asObservable')
    at get listen$ (@nestjs/core/helpers/http-adapter-host.js:47)
    at OnEventExplorer.tryRegisterHandler (on-event.explorer.js:52)
```

## Root Cause (0.10.6 fix is incomplete)

In `dist/consumer/decorators/on-event.explorer.js` line 52:

```javascript
const methodRef = target.prototype[methodName];
if (typeof methodRef !== 'function')
    return;
```

The problem: `target.prototype[methodName]` **invokes the getter** before the `typeof` guard can execute. When iterating over `Object.getOwnPropertyNames(prototype)`, it includes getter properties like `listen$` on `HttpAdapterHost`. Accessing `HttpAdapterHost.prototype.listen$` triggers the getter, which tries to read `this._listen$.asObservable()` — but `this._listen$` is undefined on the prototype (it's only set in the constructor).

The `0.10.6` guard only handles cases where `methodRef` is `undefined` after access. It does NOT handle getters that throw during access.

## Correct Fix

Use `Object.getOwnPropertyDescriptor` to inspect the property without invoking getters:

```typescript
const descriptor = Object.getOwnPropertyDescriptor(target.prototype, methodName);
if (!descriptor || typeof descriptor.value !== 'function') return;
const methodRef = descriptor.value;
const metadata = this.deps.reflector.get(ON_EVENT_METADATA, methodRef);
```

This avoids invoking getters entirely — we only look at the property descriptor's `value` field, which is only present for data properties (not accessors).

Same fix needed in `OnRequestReplyExplorer` if it has the same pattern.
