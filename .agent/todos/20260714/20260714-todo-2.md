# Fix: @cobranza-apps/events-toolkit — Add JetStream stream auto-creation

## Problem

When the library's consumer subscribes to a subject via `jetStream.subscribe()`, NATS JetStream throws `Error: no stream matches subject` if no stream is configured for that subject.

**Error in ms-db-gateway:**

```
Error: no stream matches subject
    at JetStreamClientImpl
    subject: company.*.response.v1
```

## Root Cause

NATS JetStream requires streams to be pre-defined with subject filters. The library assumes streams already exist on the server. When they don't, `js.subscribe()` fails.

## Proposed Fix

Add stream auto-creation logic in `JetStreamConsumerService.subscribe()` (or `ConsumerService`):

1. Before calling `jetStream.subscribe(subject, opts)`, check if a stream exists that covers the subject
2. Use `connection.jetstreamManager()` to query existing streams
3. If no matching stream exists, programmatically create one:

   ```typescript
   const jsm = await connection.jetstreamManager();
   const streams = await jsm.streams.list().next();
   const hasMatchingStream = streams.some(s => subjectsMatch(s.config.subjects, subject));
   
   if (!hasMatchingStream) {
     await jsm.streams.add({
       name: `auto-${subject.replace(/[^a-zA-Z0-9]/g, '-')}`,
       subjects: [subject],
       retention: RetentionPolicy.Limits,
       storage: StorageType.File,
       max_msgs: -1,
       max_bytes: -1,
       max_age: 0,
     });
   }
   ```

4. Make auto-creation configurable (opt-in via `EventsToolkitModuleOptions.consumer.autoCreateStreams`)

## Alternative: Document stream setup requirement

If auto-creation is too complex or risky, document the required NATS streams in the library README:

```bash
nats stream add EVENTS --subjects "company.*.*.v1" --storage file --retention limits
```
