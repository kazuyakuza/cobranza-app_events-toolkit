# Fix: `RequestReplyConsumerService` missing `autoCreateStreams` support [DONE]

## Problem

`@cobranza-apps/events-toolkit@0.11.1` added JetStream stream auto-creation (`consumer.autoCreateStreams`) to `JetStreamConsumerService` only. The `RequestReplyConsumerService` was **not** updated, so it still fails with `Error: no stream matches subject` when subscribing to its default `responseSubjectPattern` (`company.*.response.v1`).

**Error observed in `ms-db-gateway`:**

```
Error: no stream matches subject
  subject: company.*.response.v1
```

## Root Cause

From the compiled library (`node_modules/@cobranza-apps/events-toolkit/dist/`):

1. `consumer-module.providers.js`:
   - `createSyncRequestReplyConsumerDepsProvider()` does **not** pass `connection` or `autoCreateStreams`
   - `createAsyncRequestReplyConsumerDepsProvider()` does **not** pass `connection` or `autoCreateStreams`

2. `request-reply-consumer.service.js`:
   - Constructor receives only `jetStream`, `logger`, `responseSubjectPattern`, `dlqSubjectBuilder`
   - `onModuleInit()` calls `this.subscribe(this.responseSubjectPattern)` directly
   - `subscribe()` calls `jetStream.subscribe(subject, ...)` without any stream-existence check

3. `jetstream-consumer.service.js` (correctly implemented):
   - Constructor receives `connection` and `autoCreateStreams`
   - Creates `StreamAutoCreator` when `autoCreateStreams` is enabled
   - `subscribe()` calls `ensureStreamIfNeeded()` before `jetStream.subscribe()`

## Required Changes in Library

### 1. Update `RequestReplyConsumerDeps` interface

Add `connection?: NatsConnection` and `autoCreateStreams?: boolean` to `RequestReplyConsumerDeps`.

### 2. Update `RequestReplyConsumerService`

- Accept `connection` and `autoCreateStreams` in constructor
- Instantiate `StreamAutoCreator` when `autoCreateStreams && connection`
- Call `ensureStreamExists()` in `subscribe()` before `jetStream.subscribe()`

### 3. Update provider factories

- `createSyncRequestReplyConsumerDepsProvider()` — pass `connection` and `autoCreateStreams`
- `createAsyncRequestReplyConsumerDepsProvider()` — pass `connection` and `autoCreateStreams` from `combined.connection` / `combined.moduleOptions`

### 4. Update `ConsumerModule.forRoot()`

Pass `connection` and `autoCreateStreams` to `createSyncRequestReplyConsumerDepsProvider()`.

### 5. Tests

- Add unit tests for `RequestReplyConsumerService` verifying:
  - Stream auto-creation when `autoCreateStreams: true` and stream missing
  - No auto-creation when `autoCreateStreams: false`
  - No auto-creation when `autoCreateStreams: true` but stream already exists
- Update existing tests if they mock deps differently

### 6. Documentation

- Update CHANGELOG and any other related documentation.
