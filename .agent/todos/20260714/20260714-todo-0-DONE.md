# Fix: @cobranza-apps/events-toolkit — Runtime errors in forRootAsync path

## Background

After fixing the DI resolution bugs (missing exports, missing DiscoveryModule import, missing RequestReplyService provider), the library's `forRootAsync` path now encounters runtime errors during module initialization.

**Confirmed affected consumer:** `ms-db-gateway`.

## Errors Found

### Error 1: `Cannot read properties of undefined (reading 'ack_policy')`

**Stack trace:**

```
TypeError: Cannot read properties of undefined (reading 'ack_policy')
    at JetStreamClientImpl._processOptions (nats/jetstream/jsclient.ts:582)
```

**Context:** Logged by `EventLoggerService` as event processing error with `subject: "company.*.response.v1"`.

**Root cause:** `JetStreamConsumerService.subscribe()` calls `this.jetStream.subscribe(options.subject, options.consumerOpts ?? {})`. When `consumerOpts` is `{}` (empty object), the NATS library `JetStreamClientImpl.subscribe()` expects a `ConsumerOptsBuilder` or an object with a `config` property containing `ack_policy`. Passing `{}` causes the library to access `undefined.ack_policy`.

**Fix:** Ensure `JetStreamConsumerService.subscribe()` always passes valid consumer options to `jetStream.subscribe()`. Use the NATS library's `consumerOpts()` builder or provide a default config with `ack_policy` set.

### Error 2: `TypeError` at `Reflect.getMetadata` in `OnEventExplorer`

**Stack trace:**

```
TypeError
    at Reflect.getMetadata (reflect-metadata/Reflect.js:367)
    at Reflector.get (@nestjs/core/services/reflector.service.js:39)
    at OnEventExplorer.tryRegisterHandler (on-event.explorer.js:53)
```

**Root cause:** `OnEventExplorer.registerInstanceHandlers()` iterates over `Object.getOwnPropertyNames(prototype)`, which includes getter/setter accessor properties in addition to methods. For accessor properties, `target.prototype[methodName]` returns `undefined` (getter invoked with `this = prototype` returns undefined). `Reflect.getMetadata` throws when given an undefined target.

**Fix:** Add a guard in `OnEventExplorer.tryRegisterHandler()`:

```typescript
const methodRef = target.prototype[methodName];
if (typeof methodRef !== 'function') return;
const metadata = this.deps.reflector.get(ON_EVENT_METADATA, methodRef);
```

Same fix likely needed in `OnRequestReplyExplorer`.
