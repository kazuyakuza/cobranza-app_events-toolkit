# Fix: StreamAutoCreator must support maxBytes for NATS accounts with mandatory max_bytes

## Context

The `events-toolkit@0.11.2` fix for `RequestReplyConsumerService` `autoCreateStreams` (per `20260714-todo-4-lib-fix-requestreply-autocreate.md`) now correctly attempts to auto-create JetStream streams. However, some NATS server accounts are configured to require `max_bytes` on every stream configuration.

When `StreamAutoCreator` creates a stream without `max_bytes`, the NATS server returns:

```
NatsError: account requires a stream config to have max bytes set
  code: 400, err_code: 10113
```

This prevents `ms-db-gateway` from starting with `npm run dev`.

## Problem in Library

The `StreamAutoCreator` class (used by both `JetStreamConsumerService` and `RequestReplyConsumerService`) creates streams with a default configuration that does not include `maxBytes`. The `ConsumerModuleOptions` interface does not expose a `maxBytes` option for consumers.

## Task: Add support to StreamAutoCreator and ConsumerModuleOptions be able to set any NATS config, specially maxBytes

### Required Changes

1. Update `ConsumerModuleOptions` interface
   - Add `consumer.maxBytes?: number` option
   - and any other configuration option

2. Update `StreamAutoCreator`
   - Accept `maxBytes` (other options) in constructor or options
   - When `maxBytes`, or any other option, is provided, include `max_bytes` (or other options) in the NATS stream config passed to `jetstreamManager.streams.add()`

3. Update provider factories
   - `createSyncJetStreamConsumerDepsProvider()` — pass NATS configs from module options
   - `createAsyncJetStreamConsumerDepsProvider()` — pass NATS configs from module options
   - `createSyncRequestReplyConsumerDepsProvider()` — pass NATS configs from module options
   - `createAsyncRequestReplyConsumerDepsProvider()` — pass NATS configs from module options

4. Update `ConsumerModule.forRoot()` / `forRootAsync()`
5. Update `JetStreamConsumerService` and `RequestReplyConsumerService`
6. Tests
   - Add unit tests for `StreamAutoCreator` verifying config options are included when configured
   - Update existing consumer service tests if they instantiate `StreamAutoCreator`
7. Important: make it clear if the NATS+JetStream server blocks config options set, for auth/permissions or any error type.
8. Review logs to INFO log when sending to NATS server configuration options.
9. Update changelog
10. Define or update documentation with a clear details about the nats configuration options.
