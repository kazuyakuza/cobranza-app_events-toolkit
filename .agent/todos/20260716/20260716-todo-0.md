# Fix: `createDefaultConsumerOpts` missing `deliverTo` for push consumers

## Context

After upgrading `@cobranza-apps/events-toolkit` to `0.11.3` (which fixed the `maxBytes` issue), `ms-db-gateway` now fails to start with:

```
Error: push consumer requires deliver_subject
    at JetStreamClientImpl.<anonymous> (node_modules/nats/jetstream/jsclient.ts:497:13)
```

This happens when `RequestReplyConsumerService` or `JetStreamConsumerService` attempts to subscribe to a JetStream subject.

## Root Cause

The `events-toolkit` library's `subscribe-options.interface.ts` creates default consumer options using:

```typescript
function createDefaultConsumerOpts() {
    return consumerOpts().manualAck().ackExplicit();
}
```

This creates a **push consumer** configuration (which is what `jetStream.subscribe()` expects), but does **not** set `deliver_subject` via `.deliverTo()`.

In NATS 2.29.3, `jetStream.subscribe()` validates:

```js
if (!cso.isBind && !cso.config.deliver_subject) {
    throw new Error("push consumer requires deliver_subject");
}
```

So when `StreamAutoCreator` successfully creates the stream, the subsequent consumer subscription fails because `deliver_subject` is missing.

## Tasks

### 1. Update `subscribe-options.interface.ts` [DONE]

```typescript
import { consumerOpts, createInbox } from 'nats';

function createDefaultConsumerOpts() {
    return consumerOpts().manualAck().ackExplicit().deliverTo(createInbox());
}
```

This sets a unique inbox as the `deliver_subject` for each push consumer.

### 2. Update `resolveConsumerSubscribeOpts` [DONE]

When caller-provided `consumerOpts` already has `deliverTo` set, preserve it. When not set, ensure the default includes it.

### 3. Tests

- Add unit tests for `subscribe-options.interface.ts` verifying:
  - `createDefaultConsumerOpts()` returns a builder with `deliver_subject` set
  - `resolveConsumerSubscribeOpts(undefined)` includes `deliver_subject`
  - `resolveConsumerSubscribeOpts(existingOpts)` preserves existing `deliver_subject`

- Update changelog and related documentation
