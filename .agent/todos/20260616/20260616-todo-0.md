### 2. Outbox Transactional Hook [DONE]

Purpose:  
Ensure atomicity between database operations and saving an event to the Outbox. This prevents situations where an event is published even though the related database transaction failed (or the opposite).

Detailed Requirements:

- Add a dedicated method in `OutboxService`:

  ```ts
  async saveInTransaction<T>(
    event: EventEnvelope<T>, 
    subject: string, 
    transactionContext: TransactionContext
  ): Promise<void>
  ```

- Define a `TransactionContext` interface / type that supports:
  - TypeORM `QueryRunner` (primary use case for `ms-db-gateway`)
  - Future extensibility for other ORMs or connection types

- Expected Behavior:

  - For `ms-db-gateway` (PostgreSQL + TypeORM):
    - The event must be inserted into the `outbox` table inside the same database transaction as the business data changes.
    - If the transaction is committed → the event is saved and will be processed later by the background processor.
    - If the transaction is rolled back → the event is never saved.

  - For other microservices (SQLite):
    - The method should still work consistently.
    - Since these services usually only have a local SQLite outbox (no other business tables in the same DB), full atomicity is less critical, but the API should remain uniform.

- Provide easy-to-use helpers, for example:

  ```ts
  // Preferred usage in ms-db-gateway
  await this.outboxService.saveInTransaction(event, subject, queryRunner);
  ```

- Optionally (nice-to-have):
  - Support auto-detection of active transaction when possible.
  - Consider a decorator like `@TransactionalOutbox()` for cleaner usage (if feasible with NestJS + TypeORM).

Documentation Requirements:

- Clearly document the difference between:
  - `saveToOutbox(event, subject)` → Normal (non-transactional) usage
  - `saveInTransaction(event, subject, transactionContext)` → Transactional usage
- Provide complete code examples for `ms-db-gateway` using TypeORM transactions.
- Explain when to use each method:
  - Use `saveInTransaction` whenever a microservice performs database changes and emits one or more events as part of the same business operation.
  - Other microservices can mostly use the regular `saveToOutbox`.
