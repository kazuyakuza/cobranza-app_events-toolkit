# TODO List #2 – Producer & Consumer Modules  

Objective: Implement the Producer and Consumer layers with proper NestJS module structure, decorators, and integration with NATS/JetStream.

## Acceptance Criteria

- Both `ProducerModule` and `ConsumerModule` can be imported and configured in a NestJS microservice
- Decorators work as expected
- Messages are correctly validated on consumption
- `EventConsumerException` is properly thrown and handled for DLQ routing
- Request-Reply pattern works end-to-end (basic test)
- All services are properly injectable and testable
- Good JSDoc + comments

## Important Notes for the Implementer

- Use async/await everywhere.
- Make modules configurable (for JetStream options, connection, etc.).
- Prefer functional helpers alongside classes where it makes sense.
- Keep the library lightweight — avoid heavy dependencies.

## Tasks

### 1. Producer Module

- Create `src/producer/producer.module.ts`
- Create `src/producer/producer.service.ts` with the following public methods:
  - `publish(subject: string, event: EventEnvelope<any>): Promise<void>`
  - `emit<T>(subject: string, eventData: T, context: EventContext): Promise<void>` (convenience method)
- Inject and use `EventLoggerService`
- Handle connection to JetStream (accept `JetStream` or `NatsConnection` via constructor / module config)

### 2. Consumer Module

- Create `src/consumer/consumer.module.ts`
- Create `src/consumer/consumer.service.ts`
- Create `src/consumer/jetstream-consumer.service.ts` (handles subscription logic, message parsing, validation, and ack/nack)

### 3. Decorators

- Create `src/producer/decorators/emit-event.decorator.ts`
- Create `src/consumer/decorators/on-event.decorator.ts` – should allow easy registration like:

     ```ts
     @OnEvent('payment.proof.uploaded')
     async handleProofUploaded(event: ### PaymentProofUploadedEvent) { ... }
     ```

### 4. Request-Reply Support

- Create `src/request-reply/request-reply.service.ts`
- Create `src/request-reply/request-reply.types.ts`
- Implement helper methods:
  - `request<T, R>(subject, payload, timeout?)`
  - `sendResponse(correlationId, responseEvent)`

### 5. Validation & Error Handling Integration

- Ensure that when consuming messages:
  - The payload is transformed using `class-transformer`
  - Validated using `class-validator`
  - If validation fails → throw `EventConsumerException` (which should be caught and sent to DLQ)
- Implement proper error handling flow in `JetStreamConsumerService`

### 6. Context Helper

- Create a simple `EventContext` interface / class in `src/common/envelope/` to standardize:

     ```ts
     {
       companyId: string;
       actorType: ActorType;
       actorId: string;
       correlationId?: string;
       causationId?: string;
       traceId?: string;
     }
     ```

### 7. Update Main Barrel

- Extend `src/index.ts` to export everything from producer, consumer, request-reply, etc.
