## Tasks

### 1. Outbox Module – Configurable Design

- Create `src/outbox/outbox.module.ts` with support for multiple storage strategies.
- Define an interface `OutboxRepository`:

     ```ts
     export interface OutboxRepository {
       save(event: EventEnvelope<any>, subject: string, metadata?: any): Promise<void>;
       getPending(limit?: number): Promise<OutboxEntry[]>;
       markAsSent(id: string): Promise<void>;
       markAsFailed(id: string, error: string): Promise<void>;
     }
     ```

- Implement two concrete repositories:
  - `SqliteOutboxRepository` (default for most microservices)
  - `PostgresOutboxRepository` (for `ms-db-gateway`)
- Make the module configurable via `EventsToolkitModule.forRoot({ outbox: { type: 'sqlite' | 'postgres', ...options } })`
- Create `src/outbox/sqlite-outbox.repository.ts`
- Create `src/outbox/postgres-outbox.repository.ts` (lightweight, assumes TypeORM)
