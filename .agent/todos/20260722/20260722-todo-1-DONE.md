# TODO: Extend `@cobranza-apps/events-toolkit` to Support Durable JetStream Consumers

## Context

The `ms-db-gateway` service uses `@cobranza-apps/events-toolkit` (v0.13.0) for NATS JetStream consumption. When the gateway reconnects to NATS (e.g., after a restart), it receives and re-processes all historical events because the toolkit creates an **ephemeral push consumer** without a `durable_name`. On disconnect, the ephemeral consumer is destroyed; on reconnect, a new ephemeral consumer is created with `DeliverPolicy.All` (default), replaying the entire stream.

## Problem

The toolkit's `EventsToolkitConsumerOptions` interface only exposes:

- `enable?: boolean`
- `dlqSubjectBuilder?: (subject: string) => string`
- `autoCreateStreams?: boolean`
- `streamConfig?: Partial<StreamConfig>`

It does **not** expose JetStream **consumer-level** options such as `durable_name`, `ack_policy`, `deliver_policy`, `max_deliver`, or `replay_policy`. Therefore, the gateway cannot configure a durable consumer through the toolkit's public API.

## Tasks

### 1. Extend `EventsToolkitConsumerOptions` [DONE]

Add optional consumer-level configuration fields to the interface:

```ts
export interface EventsToolkitConsumerOptions {
  enable?: boolean;
  dlqSubjectBuilder?: (subject: string) => string;
  autoCreateStreams?: boolean;
  streamConfig?: Partial<StreamConfig>;
  // NEW:
  consumerOpts?: Partial<ConsumerOpts> | ConsumerOptsBuilder;
  durableName?: string;
  deliverPolicy?: DeliverPolicy;
  ackPolicy?: AckPolicy;
  maxDeliver?: number;
  replayPolicy?: ReplayPolicy;
}
```

Or, preferably, accept a `consumerOpts` builder/callback that gives full control:

```ts
consumerOptsBuilder?: (subject: string) => ConsumerOptsBuilder;
```

### 2. Pass Consumer Options Through to Subscription [DONE]

Update the following internal services to pass the configured consumer options to `jetStream.subscribe()`:

- `JetStreamConsumerService.subscribe()`
- `RequestReplyConsumerService.subscribe()`

Currently, these services call `resolveConsumerSubscribeOpts()` which defaults to `consumerOpts().manualAck().ackExplicit().deliverTo(createInbox())`. When gateway-level `durableName` or `deliverPolicy` is provided, these must be merged into the builder before subscription.

### 3. Ensure Durable Consumer Resume on Reconnect [DONE]

When `durableName` is provided:

- The same `durable_name` must be used on every reconnect.
- NATS server will persist the consumer's last acknowledged position.
- The consumer must resume from that position instead of replaying history.
- If `deliverPolicy` is omitted when `durableName` is set, NATS will use the durable's stored state automatically.

### 4. Update Type Exports [DONE]

Ensure the new options are exported from the public API (`index.d.ts`) so the gateway can import and use them.

### 5. Add Tests [DONE]

Add unit/integration tests verifying:

- `durableName` is passed to `jetStream.subscribe()`.
- `deliverPolicy` is respected.
- Ephemeral consumer remains the default when no `durableName` is provided.
- and same for all new available configurations
