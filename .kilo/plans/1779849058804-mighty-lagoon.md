# Plan: Fix Critical Workflow Sub-Agent Type Assignment

## Problem
The Plan Agent assigns steps 2, 3, 4.1, 4.2, 4.5, 4.6 (and 4.3 fix application) to `subagent_type: "general"` instead of the correct types. This happens because `critical-workflow.md` uses ambiguous terms ("Code sub-agent", "Plan sub-agent") without ever specifying the exact `subagent_type` parameter value for the `task` tool.

## Root Cause
`code` and `plan` are **built-in Kilo agents** (see [docs](https://kilo.ai/docs/code-with-ai/agents/using-agents#built-in-agents)). They ARE valid `subagent_type` values. The workflow document simply never maps its terminology to these values. The Plan Agent reads "assign to Code sub-agent" and, lacking a specific `subagent_type`, defaults to `"general"`.

| Workflow Term | Built-in/Defined `subagent_type` | Currently Used |
|---|---|---|
| "Code sub-agent" | `code` (built-in) | `general` ❌ |
| "Plan sub-agent" | `plan` (built-in) | `general` ❌ |
| "Code Reviewer sub-agent" | `code-reviewer` (`.kilo/agents/`) | `general` ❌ |
| "Docs Specialist sub-agent" | `docs-specialist` (`.kilo/agents/`) | `general` ❌ |

## Fix: Single Component

**Update `.kilo/commands/critical-workflow.md`** to add explicit `subagent_type` parameter values in every delegation instruction and in the example.

No new agent files needed — `code` and `plan` are built-in, `code-reviewer` and `docs-specialist` already exist in `.kilo/agents/`.
