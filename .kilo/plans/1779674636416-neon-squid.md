# Plan: Tool Selection Priority Rule

**Date:** 2026-05-24  
**Goal:** Create a general rule requiring AI agents to evaluate all available tools (including MCP tools) before choosing which to use for a given operation.

## Problem

Agents default to bash/console commands for operations where MCP tools provide better semantics (e.g., rename with import-refactoring, search with semantic understanding). No existing rule directs agents to survey their full tool inventory before acting.

## Implementation Steps

### Step 1: Create `.kilo/rules/tool-selection-priority.md`

New rule file with the following content:

```markdown
# Tool Selection Priority Rule

- Before using any tool, evaluate the FULL set of available tools — including all MCP-provided tools — for the operation at hand.
- Prefer tools with semantic understanding of the codebase (e.g., `vscode-mcp-server_rename_file_code`, `vscode-mcp-server_move_file_code`, `Bifrost_*`) over raw file or text commands for code operations.
- Prefer structured editors (`vscode-mcp-server_replace_lines_code`, `vscode-mcp-server_create_file_code`) over plain `edit` for code changes.
- Reserve `bash` for CLI-native operations (git, npm, builds, tests) — not for file manipulation or code refactoring.
- Fall back to `bash` or `edit` only when no semantic MCP tool covers the operation.
```

### Step 2: Update `.agent/RULES.md`

Add the new rule to the rules list:

- Insert after the existing rules list entry: `- [Tool Selection Priority](../.kilo/rules/tool-selection-priority.md)`

Position: After `[Code Guidelines](../.kilo/rules/code-guidelines.md)` entry (last in current list), maintaining alphabetical/logical grouping.

### Step 3: No Changes to Agent Prompts

The rule applies globally via `.agent/RULES.md` linkage. Individual agent prompts in `.kilo/agents/*.md` do not need modification since they all load rules automatically.
